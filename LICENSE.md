## 🛠️ Troubleshooting Common Issues

- ❌ **Script does not execute after injection**
  The executor may not be properly attached to the Roblox process. Close both applications completely, relaunch Roblox first, wait until you are fully loaded into the Sword Factory map, then open your executor and try again.
  **Solution:** Always attach after the game world has fully rendered. Update your executor to the latest version if the problem continues.

- ❌ **GUI does not appear in-game**
  This usually happens when another script's interface conflicts with the overlay layer. A corrupted config file can also prevent the menu from drawing.
  **Solution:** Delete your `config.lua` at `...Documents/SFScript/config.lua`, re-execute the Sword Factory Script, and let it regenerate a fresh default config.

- ❌ **Auto Farm stops after a few minutes**
  Roblox occasionally shifts ore spawn coordinates in minor patches, causing the pathing module to loop on empty nodes.
  **Solution:** Pull the newest version of the Sword Factory Script Pastebin paste — path data is updated with every release.

- ❌ **Kicked from the server unexpectedly**
  Using extreme speed or jump values in public lobbies increases visibility. Other players may report unusual movement.
  **Solution:** Lower `walkSpeed` to `32` or below and keep `jumpPower` under `120`. Use these features conservatively in busy servers.

- ❌ **Mobile executor crashes on inject**
  Some Android executors struggle with large script payloads on devices with limited RAM.
  **Solution:** Close all background apps, clear the executor cache, and retry. If it persists, switch to a lighter executor build.
